Features

Everything you need.
Nothing you don't.

Built over 15 development phases, OpenComputer is a complete desktop AI agent with real tools, persistent memory, and a premium interface.

💬
Streaming Chat
Real-time SSE streaming with any OpenAI-compatible provider. iMessage-style bubbles with reactions, threading, and message feedback.
🧠
Memory Store
SQLite FTS5 full-text search for facts and knowledge. Truth Vault, cross-session state, and user corrections — all persisted locally.
🔍
Web Search
Multi-engine search (DuckDuckGo, Google, Bing, SearXNG). HTML→Markdown scraping. Crawl with depth control. Search history with favorites.
📁
File Operations
Read, write, edit, list, move, delete files — all sandboxed with path traversal protection. Drag-and-drop attachments in chat.
💻
Terminal
Sandboxed subprocess execution with command whitelisting. Risk classification for high-danger commands. Full terminal emulator UI.
🌿
Git Integration
Init, commit, push, pull, branch, diff, PR — all through the Rust backend. Push requires confirmation. Branch switching warns on uncommitted changes.
🤖
Subagent Manager
Create custom AI subagents with their own system prompts, tool permissions, and task type mappings. Enable/disable on demand. Test with live models.
🔌
MCP & Plugin Manager
Browse 15+ MCP servers (Slack, Gmail, GitHub, etc.), one-click install, configure env vars, enable/disable, start/stop. Custom MCP server support.
⚡
Background Tasks
Async task execution with SQLite-backed tracking. Monitor progress, cancel tasks, view history. APScheduler for cron-based recurring work.
🔀
Model Merging
Sens-Merging, Activation-Informed Merging, Dynamic Merging. Optimizer auto-selects the best strategy per task type based on performance history.
📦
Compact Context
Model-driven context compaction with compact, discard, protect, and prune tools. Auto-compact at 80% context usage. Never lose important context.
🎨
Floating Glass 3D
Apple Liquid Glass style interface with domed convex panels, gradient rim-light borders, animated blobs, and Framer Motion spring gestures.
🔐
Local-First
All data stored locally in SQLite. Provider credentials in Tauri's encrypted store. No cloud dependency. No telemetry. No lock-in.
Architecture

Built to be
understood.

A clean separation between UI, native shell, agent backend, and tools. Every layer is replaceable.

System Overview
Three-Layer Architecture
┌──────────────────────────────────────────────────────────────┐ │ React UI ←→ Tauri Rust Shell ←→ SQLite + Filesystem │ └──────────────────────────────────────────────────────────────┘ ↕ ↕ ┌─────────────────────┐ ┌───────────────────────────────────┐ │ PHAOS Backend │ │ MCP Tools │ │ (Python FastAPI) │ │ File · Terminal · Git · Web · DB │ │ Port 8420 │ │ Webhooks · Cron · Search │ └─────────────────────┘ └───────────────────────────────────┘ ↕ ┌──────────────────────────────────────────────────────────────┐ │ Any OpenAI-Compatible LLM Provider │ │ (Cerebras, Groq, OpenAI, Anthropic, Ollama, etc.) │ └──────────────────────────────────────────────────────────────┘
Frontend
React + TypeScript
Strict-mode TypeScript. 80+ components. Custom hooks for chat, memory, search, reactions, threads, and subagents. Framer Motion spring animations throughout.
src/
Native Shell
Tauri v2 + Rust
Native desktop shell with SQLite, file system access, git operations, and auto-update. Store plugin for encrypted credential persistence.
src-tauri/
Agent Backend
PHAOS (Python)
ReAct agent loop with state machine, tool router, error recovery, and circuit breaker. Memory, self-teaching, streaming, and scheduling.
phaos/
Tools
MCP Tool Layer
Sandboxed file, terminal, git, web, database, webhooks, cron, and search tools. All calls go through permission checks. Web search across 4 engines.
phaos/routes/
